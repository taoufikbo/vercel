Notions Fondamentales

14 définitions opérationnelles + analogie — pour lire le deck sans prérequis ML

LLM
Large Language Model
Modèle entraîné sur des milliards de textes pour prédire le token suivant. Capable de générer, résumer, classer. Ex : GPT-4, Granite, Mistral.
Un stagiaire qui a lu toute la bibliothèque — mais qui peut halluciner avec confiance.
Embedding
Représentation vectorielle sémantique
Transformation d'un texte en vecteur numérique. Deux phrases sémantiquement proches ont des vecteurs proches. Base du RAG.
GPS du sens : 'Panne Nokia' et 'défaillance équipement Nokia' sont proches dans l'espace vectoriel.
RAG
Retrieval-Augmented Generation
Architecture combinant recherche vectorielle + génération LLM. On interroge une base de documents en temps réel et on injecte les passages pertinents dans le prompt.
Le LLM avec accès à une bibliothèque consultable en live — vs le LLM qui ne sait que ce qu'il a mémorisé.
Fine-tuning
Ajustement fin sur données domaine
Ré-entraînement d'un LLM sur un dataset métier spécifique. Coûteux mais produit un modèle qui parle le vocabulaire du domaine.
Prendre un généraliste diplômé et lui faire faire une spécialisation telco de 6 mois sur vos cas réels.
Training
Entraînement du réseau de neurones
Optimisation des poids sur un grand dataset. Pré-training = données générales (mois, GPU clusters). Fine-tuning = adaptation domaine (heures/jours).
Le pré-training est le bac général. Le fine-tuning est la spécialisation. L'un sans l'autre ne fait pas un expert métier.
Inference
Génération en production (serving)
Phase d'utilisation du modèle entraîné. Poids figés, inputs → outputs. KServe + vLLM servent l'inférence. Métrique clé : TTFT et tokens/sec.
Training = apprendre à conduire. Inference = conduire au quotidien avec le permis.
Quantization
Compression des poids numériques
Réduction FP16 → INT8/INT4. Réduit l'empreinte VRAM et accélère l'inférence. Perte qualité généralement < 2%.
Passer d'un FLAC audio à un MP3 bien encodé — l'oreille (le modèle) ne voit presque pas la différence.
MLOps
DevOps appliqué au cycle de vie ML
Pratiques (GitOps, CI/CD, monitoring, versioning) appliquées aux modèles pour garantir reproductibilité et fiabilité en production.
Ce qui sépare un modèle qui marche dans un notebook d'un modèle qui délivre de la valeur à 3h du matin.

12 Briques Analysées sur 6 Axes

Chaque brique couvre : Historique · Utilité · Limites · Concurrent proprio · Concurrent OSS · Prérequis OCP

Data Engineering

Docling

Document chunking & pre-processing. Ingestion universelle PDF/DOCX/HTML pour pipeline RAG. IBM Research, open source MIT.

Concurrent : Azure Document Intelligence, Unstructured.io
Model Development

JupyterLab Workbench

Notebook-as-a-Service managé sur OCP. GPU scheduling, bibliothèques AI/ML pré-intégrées, multi-user RBAC.

Concurrent : Databricks Notebooks, SageMaker Studio
Model Training

InstructLab

Fine-tuning RLHF synthétique (méthode LAB, MIT × Red Hat). Génère des données d'entraînement sans annotateurs massifs.

Concurrent : Axolotl, LLaMA-Factory, OpenRLHF
Model Training

Ray + Kubeflow Training Op.

Distributed training FSDP sur N GPUs. Ray Tune pour hyperparameter search. PyTorchJob déclaratif Kubernetes.

Concurrent : DeepSpeed, Megatron-LM, NVIDIA DGX Cloud
Model Optimization

llm-compressor

Quantization INT8/FP8, pruning, distillation. Neural Magic (acq. Red Hat 2024). Compatible vLLM post-compression.

Concurrent : GPTQ, AWQ, TensorRT-LLM
Model Evaluation

GuideLLM + TrustyAI Eval

Benchmarking performance (TTFT, P99) + évaluation qualité et biais. Conformité AI Act EU intégrée.

Concurrent : lm-evaluation-harness, RAGAS, DeepEval
Model Deployment

KServe

Inference gateway Kubernetes multi-runtime. Canary, A/B testing, autoscaling KEDA. CNCF sandbox.

Concurrent : Ray Serve, BentoML, Seldon Core
Model Deployment

vLLM

Le standard de facto OSS pour serving LLM haute performance. PagedAttention, multi-LoRA, OpenAI-compatible API.

Concurrent : TGI (HuggingFace), TensorRT-LLM, LMDeploy
Model Deployment

OpenVINO + llm-d

Intel OpenVINO pour inférence CPU/edge. llm-d : prefill/decode disaggregation pour vLLM multi-nœuds OCP.

Concurrent : llama.cpp, Ollama, MLC-LLM
MLOps

ArgoCD + Kubeflow Pipelines

GitOps ML complet. Pipeline CI/CD pour modèles, reproductibilité totale via Git. Argo Workflow pour DAG training.

Concurrent : Metaflow, ZenML, SageMaker Pipelines
MLOps

Feast + Model Registry

Feature Store point-in-time correct + catalogue versionné des modèles (champion/challenger, audit trail).

Concurrent : MLflow Registry, Databricks Feature Store
Gouvernance

TrustyAI Guardrail

Détection biais, explainability SHAP/LIME, LLM guardrail (Granite Guardian). Conformité AI Act EU Art. 9.

Concurrent : Evidently AI, Langfuse, AWS Bedrock Guardrails
